# Sistema-de-control-de-usuarios-y-roles

El funcionamiento del sistema comienza cuando un usuario accede a la página de inicio de sesión (login). En este formulario el usuario introduce su nombre de usuario o correo y su contraseña. Cuando se envían estos datos, el archivo PHP correspondiente recibe la información y realiza una consulta a la base de datos para verificar si existe un usuario con esas credenciales. Si los datos son correctos, el sistema crea una sesión en PHP, la cual guarda información como el nombre del usuario y su rol dentro del sistema. Esta sesión permite que el sistema recuerde que el usuario ha iniciado sesión mientras navega por las diferentes páginas.

Después de iniciar sesión, el sistema redirige al usuario a un panel o página principal. En este punto el sistema revisa el rol del usuario guardado en la sesión para determinar qué acciones puede realizar. Si el usuario tiene el rol de administrador, el sistema le permite acceder a todas las funciones del sistema, como crear, editar o eliminar usuarios y cursos. En cambio, si el usuario tiene el rol de usuario normal, solo podrá visualizar la información disponible, como la lista de cursos, pero no podrá realizar modificaciones en los datos.

El sistema también incluye un módulo de gestión de usuarios, donde el administrador puede realizar las operaciones básicas conocidas como CRUD (Create, Read, Update, Delete). Esto significa que el administrador puede crear nuevos usuarios, ver la lista de usuarios registrados, editar su información o eliminarlos del sistema. Estas acciones se realizan mediante formularios que envían la información a archivos PHP que se encargan de ejecutar las consultas correspondientes en la base de datos.

De forma similar, el sistema tiene un módulo para la gestión de cursos. En este módulo el administrador puede crear nuevos cursos, editarlos o eliminarlos. Los cursos se almacenan en una tabla de la base de datos y luego se muestran en la interfaz del sistema mediante tablas HTML generadas dinámicamente con PHP. Los usuarios normales únicamente pueden ver la lista de cursos disponibles, lo cual asegura que no puedan alterar la información.

Para conectarse con la base de datos, el sistema utiliza un archivo de conexión que se encarga de establecer la comunicación con MySQL utilizando PHP. Este archivo se incluye en las diferentes partes del sistema para que todas las consultas a la base de datos se realicen correctamente. Además, el sistema utiliza sesiones para proteger las páginas internas, de modo que si alguien intenta acceder directamente a una página sin haber iniciado sesión, el sistema lo redirige nuevamente al login.
